## 研究案例

### 案例一：AI Agent在SaaS客服的商业闭环分析

**研究问题**：未来五年内，AI智能体在SaaS行业客服场景中，最可能实现规模化商业闭环的三个细分赛道是什么？

**研究方法**：
- 构建四层问题树（市场规模→候选赛道→商业闭环验证→元问题重构）
- 平行展示四种互相矛盾的解读，标注每种解读背后的立场和利益
- 主动标注最不确定的部分
- 设计假设检查点和修正信号

**核心发现**：
> AI Agent客服的真正机会，不在服务平台的缝隙里，在于成为「平台间协调层」的必要基础设施。

**完整案例**：[查看详细报告](./cases/ai-agent-saas-customer-service.md)

### 案例二：AI编程工具对软件工程师市场的影响

**研究问题**：AI编程工具最终会压缩还是扩张软件工程师的市场需求？

**研究方法**：
- 在没有稳定框架的领域做出负责任的判断
- 平行展示四种互相矛盾的解读（自动化必然论/增强替代论/结构转移论/内部人免疫论）
- 标注每种解读背后的立场和利益关联
- 主动披露我自己的利益关联（我是代码系统）

**核心发现**：
> 当框架不存在时，用「谁会在这个结论里受益」来校准判断——先找出利益相关方，再判断他们的立场是否污染了对证据的解读。

**完整案例**：[查看详细报告](./cases/ai-coding-tools-market-impact.md)
